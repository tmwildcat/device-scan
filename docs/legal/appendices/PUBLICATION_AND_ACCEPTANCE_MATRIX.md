## 1. Purpose and scope

This matrix states candidate display and acceptance controls. “Re-accept” means material changes only, subject to counsel.

## 2. Matrix

| Document | Display/link | Affirmative acceptance | Version recorded | Re-accept |
|---|---|---|---|---|
| Terms of Use | public footer; registration/login notice | registration and material gated use | yes | where material/required |
| Registered User Agreement | registration/account | registration | yes | where material |
| Subscriber Agreement | checkout/account | checkout | yes | material paid-term change |
| AUP / Engineering Disclaimer | footer; upload/compare/export warnings | incorporated; focused warning acknowledgement where risk requires | linked version | material safety/use change |
| Billing and Refund Policy | checkout/account | checkout acceptance bundle | yes | changed renewal/refund where required |
| Privacy/GDPR/Cookie | footer; collection points; banner | consent only where legal basis requires | notice/consent versions | new consent/purpose as required |
| Manufacturer Agreement/policies | invitation onboarding; publication workflow | onboarding and rights grant; publication confirmation | yes | material licence/workflow change |
| Enterprise Agreement/DPA/SLA | Order package | signature/Order | executed copy | amendment/renewal as agreed |
| API and MCP Terms | credential issuance/docs | credential issuance or Order | yes | material interface/data-use change |
| Copyright/Security/Disclosure/Open Data | footer/docs/export surfaces | generally incorporated or notice; licence acceptance if required | published version | material rights change |

## 3. Acceptance record

Record person/organisation, Account, document ID/version, timestamp, method, locale, and available IP/user-agent or signature evidence. Preserve exact accepted text under the retention schedule.
